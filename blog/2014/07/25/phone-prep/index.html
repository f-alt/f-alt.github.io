Getting my phone ready for DEFCON

2014-07-25

This post was supposed to be a wonderful HOWTO on setting up your Android phone with a VPN connection that would fail shut, ie: block all non-vpn traffic out. Sadly, this mission has proven to much more difficult than I first envisioned.

I’m working under the assumption that at DEFCON (the world’s most hostile network) as a first timer, I’m going to get pwned. From that assumption, it follows that I can’t bring any device with me that I’m not willing to completely wipe. It also follows that I can’t have any way to log into any of my regular accounts, ESPECIALLY those related to work. For that reason, I’ve set up an ALTnernate identity, VPS, laptop and phone.

Now that I’ve finished all of that, I can work on trying to make it harder for me to get hacked. I’ve heard a lot people suggest that I rely on cellular data, instead of the wi-fi at DEFCON, but spoofing a cell phone base station is entirely within the realm of possibility. Instead, I want to set up a system of my devices so that the only way to get internet is through an openvpn connection. I’ve decided against running my own openvpn server, since I don’t trust my skills enough to lock it down. Instead I purchase a VPN subscription. I found a few decent looking providers like Torguard, AirVPN, WiTopia and Private Internet Acess.

My first attempts at setting things up involved OpenVPN for Android. I had a few difficulties importing the config files from my VPN provider, but eventually I got things configured. Sadly I can’t use this open source client, since 1) it doesn’t allow me to block outgoing connections if the VPN isn’t connected, and 2) IT WON’T REMEMBER MY CREDENTIALS!! It’s this second point that made me abandon this app, since I’m using a password manager and a very long and impossible to remember password which I can’t type in everytime.

Next, I tried to set up IPSec through Android native VPN interface, but that didn’t work. I almost resigned myself to using PPTP since there’s a menu item in the VPN settings that reads “Always on VPN”, but it doesn’t seem to work on Cyanogenmod 11…

I’ve settled on OpenVPN Connect for the time being, since it will connect at boot AND it remember my login details. Sadly, Android 4.3+ has this handy dandy confirmation dialogue that pops up whenever a vpn client starts asking whether you trust the app or not and it does not have an option to remember this decision…

The next thing I’m going to try and build is an iptables script that will disable all but VPN traffic.
